# dla2D

Симулятор двумерной диффузионно-ограниченной агрегации (Diffusion-Limited Aggregation, DLA).

DLA — это процесс, при котором частицы совершают случайное блуждание (брауновское движение) и, столкнувшись с растущим кластером, намертво прилипают к нему. Этот процесс образует характерные фрактальные структуры (например, снежинки, кораллы, молнии).

## Структура проекта

* `dla.c` — основной файл программы, содержащий алгоритм симуляции DLA.
* `paulslib.c` / `paulslib.h` — библиотека вспомогательных функций (математика, работа со сплайнами и т.д.).
* `bitmaplib.c` / `bitmaplib.h` — библиотека для сохранения результата растрового изображения в формат TGA.
* `Makefile` — скрипт для автоматической сборки проекта.

## Требования

* Операционная система: Linux (или любая другая с установленным `gcc` и `make`).
* Компилятор: `gcc`
* Утилиты: `make`
* Стандартная математическая библиотека: `libm`

## Как скомпилировать

Проект собирается с помощью утилиты `make`.

**Важное замечание:** В оригинальном коде мог присутствовать флаг `-arch i386` (специфичный для компилятора Apple). Если вы собираете проект на Linux, убедитесь, что в `Makefile` в переменной `CFLAGS` этого флага нет (оставьте только `-Wall -O3`).

```bash
# Очистить предыдущие сборки (если есть)
make clean

# Скомпилировать проект
make

После успешной компиляции в папке появится исполняемый файл dla.

(Примечание: при компиляции могут появляться предупреждения warning: ignoring return value of ‘fread’ в файле bitmaplib.c. На работу программы они не влияют и могут быть проигнорированы).

Как запустить
После компиляции запустите программу из терминала:

bash

./dla [параметры]
Какие возможны параметры при старте?


Обычно программы DLA принимают следующие аргументы командной строки:

-N <число> — количество частиц для симуляции (например, 10000).
-s <число> — размер сетки/поля (например, 500 пикселей).
-o <имя_файла> — имя выходного файла картинки (обязательно с расширением .tga, например, fractal.tga).
Пример запуска:

bash

./dla -N 5000 -s 800 -o result.tga
(Если запустить программу без параметров ./dla, она использует значения по умолчанию, жестко заданные в коде).

Процесс работы
После запуска программа начинает вычисления. В терминале вы будете видеть вывод вида:

text

Seed: 0 998 502
Seed: 1 836 131
Seed: 2 533 2
...
Это означает, что программа успешно стартовала и генерирует/обрабатывает начальные координаты частиц.

Важно: Алгоритм DLA сильно замедляется по мере роста кластера. Не прерывайте программу (с помощью Ctrl+C), пока она не завершится сама, если вы хотите получить корректную картинку.

Результат
После завершения работы программы (когда процесс вернется в командную строку) в папке проекта появится файл изображения в формате TGA (.tga).

Вы можете открыть его любой программой просмотра изображений, поддерживающей этот формат (GIMP, Eye of GNOME, XnView и т.д.):

bash

eog result.tga
# или
xdg-open result.tga
Решение проблем (Troubleshooting)
Ошибка gcc: error: unrecognized command-line option ‘-arch’
Решение: Откройте Makefile, найдите CFLAGS и удалите оттуда -arch i386.
Программа долго ничего не делает, но ошибки нет
Решение: Это нормальное поведение алгоритма. Чем больше частиц уже прилипло, тем дольше новой частице нужно блуждать, чтобы найти кластер. Просто подождите.
